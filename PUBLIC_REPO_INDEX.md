# Public Repo Index

Everything you need to move this prototype into a fresh public repo. Lists what to ship, what to leave behind, and what to set up on the new side so the codebase runs cleanly out of the gate.

## 1 · What to include

### Root files (mandatory)
| File | Why |
| --- | --- |

| `tailwind.config.ts` | Design tokens — colour, type scale, rounded radii |
| `postcss.config.mjs` | Tailwind / PostCSS pipeline |
| `next.config.mjs` | Next.js config |
| `next-env.d.ts` | Next.js TS shims (committed by convention) |

| `.gitignore` | Keep (will need a small edit, see §3) |
| `components.json` | shadcn/ui config (kept even though we don't use it heavily) |
| `README.md` | Replace with the public-facing one (see §4) |

### Documentation (mandatory if you want the design system to be useful)
| File | Why |
| --- | --- |
| `DESIGN_SYSTEM.md` | Tokens, type scale, status colours, 14 component recipes. The handoff cheat-sheet. |

## 4 · Suggested README for the public repo

Replace the existing `README.md` with something like:

```markdown
# Revspot — Voice + Outreach Prototype

An interactive Next.js prototype demonstrating an AI-driven outreach
workspace: voice agents that call CSV-uploaded lead lists, qualify on
the call, and feed a CRM. Built for Indian real-estate developers
(Godrej South — Bangalore + Hyderabad demo workspace).

This is **prototype-grade**. No backend, all data is mocked in
TypeScript modules under `src/lib/`. The point is to walk through the
full UX of a paid voice + outreach product, not to be production-ready.

## Run it

\`\`\`bash
npm install
npm run dev
\`\`\`

Open http://localhost:3000

## The flow

1. **`/signup`** — create an account (full name, email, password) or SSO
2. **`/get-started`** — 4-step personalisation wizard (Company → Role → Use case → First project)
3. **`/welcome`** — guided 2-step plan: create voice agent → launch outreach
4. **`/outreach`** — the workspace

## Design language

See `DESIGN_SYSTEM.md` for tokens, type scale, status colours, and 14
component recipes. See `PROTOTYPE_OVERVIEW.md` for the product surfaces
and conventions.

## Stack

- Next.js 14+ App Router
- TypeScript
- Tailwind CSS (tokens in `tailwind.config.ts`)
- framer-motion (page + step transitions)
- lucide-react (icons)
- date-fns (dates)

## License

[Your choice — MIT is common for design prototypes]
```

## 5 · Files I'd star (the "tour" for a new contributor)

If someone clones the repo and wants to feel the product in 15 minutes,
have them read these in order:

1. `PROTOTYPE_OVERVIEW.md` — product context
2. `DESIGN_SYSTEM.md` — the design language
3. `src/app/signup/page.tsx` + `src/app/get-started/page.tsx` — the
   colourful onboarding journey, demonstrates pastel palette + step rhythm
4. `src/app/(app)/outreach/page.tsx` — the listing pattern (status tabs,
   segmented filter, KPI strip, Spot Insights, paginated table)
5. `src/app/(app)/outreach/[id]/page.tsx` — the detail pattern (4-up KPI
   widgets, contacts table with status tabs, lead drill-down drawer)
6. `src/app/(app)/outreach/create/page.tsx` — the wizard pattern
   (2-step + interstitial, per-file schedule, per-file progress bars,
   inline error banner, optional test panel)
7. `src/components/project/outreach-section.tsx` — the project-tab
   pattern (header CTA + KPI strip + table + empty state)
8. `src/components/dashboard/metric-card.tsx` — the canonical KPI tile
9. `src/components/layout/sidebar.tsx` — sidebar + wallet widget
10. `src/lib/outreach-data.ts` — sample data shape

## 6 · One last sanity pass before pushing

- Search the repo for hard-coded emails, phone numbers, names, or
  workspace IDs you don't want public. Quick:
  \`\`\`bash
  grep -rE "@[a-z]+\.com|9[0-9]{9}" src/ lib/
  \`\`\`
- Audit `public/` for non-public assets (logos that aren't yours, etc.)
- Re-read `design-handoff.md` and `docs/` for client-specific content
  before publishing
- Make sure the demo data in `src/lib/*.ts` reads as realistic but
  fictional — the current Godrej / Lodha / Prestige names are real
  developers; consider renaming if that's a concern.
